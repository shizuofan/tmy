# 任务主体
生成一个桌面应用程序，用于有声小说制作
# 基本信息
## 软件名称
听墨语
## 风格
仿照adobe视频剪辑软件风格
# 软件架构
采用wails架构，前端框架为react，数据库采用sqlite3
# 注意事项
考虑依赖兼容性，尽可能使用稳定版本
# 数据模型
## 工程
包含结构：
- 工程id
- 章节列表
## 章节
包含结构：
- 文档内容
- 段落列表（音轨）
## 段落
- 文本内容
- 音色id
- 情感参数（愤怒、兴奋等等）
- 语速
## 音色
- 音色id
- 标题
- 描述
- 支持的情感参数类型
# 界面
## 首页+编辑工程列表
如果没有工程则可以创建工程，如果有工程则展示工程列表+创建工程
## 工程详情+编辑
段落：展示段落列表，可以删除，调整顺序，导入章节文本（可以导入多章节）
角色：工程中涉及的角色，人物关系，角色对应音色，可配置，角色简介
## 章节详情+编辑
仿照视频编辑软件有一个音轨，选择某个段落，高亮对应文本，可以纠正编辑音色id 情感参数等，调整段落顺序，删除段落等操作
# 主要运行流程
1. 用户创建工程，导入小说文本，调用文本大模型，获取：
  - a. 小说中角色，维护在工程内
  - b. 小说章节，按顺序维护在工程内
  - c. 小说段落（每个人说的一句话，包括旁白），并识别出哪个角色，什么语气
2. 点击章节详情，此时并没有生成音频，可以批量生成音频，调用语音合成大模型，获取：
  - a. 每个段落的音频，并存储
3. 用户试听后调整段落配置，重新生成单个段落并确认后替换
